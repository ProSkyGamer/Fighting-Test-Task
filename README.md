* Ориентировочное потраченное время - ~6-7 часов

Логика проекта построена на основе шаблона ECS, т.к. он наиболее удобен для реализации в unity и позволяет простые способы расширения для данного типа игры
В данном случае существует объект сущности (игрок или враг), внутри него существует его визуальная составляющая, а также будут существовать дальнейшие возможные компоненты свойств
Компонент свойств - (в данном случае - форма, цвет, размер) - все данные свойства наследуются от базового класса свойств. Базовый класс свойств содержит тип свойства (enum), а также доп. характеристики добавляемые к статам сущности. Если конкретное свойство содержит доп. характеристики (в данном случае визуальную составляющую) они описываются внутри специальных методов класса наследника. 
Класс лагеря сущностей - содержит поля количества спавнящихся сущностей, префабов сущностей, позиции спавна, обязательные свойства у сущностей и др.
При вызове метода спавна сущностей создается сущность из префаба, выбираются все обязательные компоненты свойств сущности и добавляются на созданную сущностей, повторяется по количеству полей сущности. 
Игровой менеджер- содержит все лагеря сущностей и др поля.
При нажатии кнопки на ui во всех лагерях создается максимальное количество сущностей и открывается возможность запустить игру.

* Доп. функция:
Бустеры- доп. способности игрока, которые он может приобретать* и использовать в нужный момент игры.
Н.: заморозка- замораживает всех врагов на определенное время, после чего уходит на кулдаун.
*Бустеры приобретаются в главном меню за монеты, которые можно получить за каждого убитого врага

* Возможности расширения: 
Добавление новых свойств сущности/новых бустеров - создание нового класса и наследование его от базового, описание доп. логики его работы в новом классе
Добавление новых лагерей врага/игрока - создание новых объектов на сцене, добавление в объект игрового менеджера
Добавление новых команд (второй команды врагов и тд) - добавление нового свойства в enum типа существ, изменение поля типа существ в любом из лагерей 
Добавление/удаление присваеваемых свойств новым создаваемым существам- добавление/удаление типа свойства в соответсвующем поле каждого лагеря
